Axolotl supports supervised fine-tuning of vision-language models (VLMs) and other multimodal architectures. Multimodal support is currently in beta and does not have full feature parity with text-only training. Most standard SFT options apply, but a few additional config keys are required and some features (such as sample packing) are not yet available.

<Warning>
Multimodal fine-tuning is a beta feature. We do not currently truncate or drop samples based on `sequence_len`, as each architecture processes non-text tokens differently.
</Warning>

## Required configuration

These keys are required for all multimodal fine-tuning runs:

```yaml multimodal-base.yml
processor_type: AutoProcessor

skip_prepare_dataset: true
remove_unused_columns: false  # columns are needed for image embedding handling
sample_packing: false         # not yet supported with multimodal

# example dataset
datasets:
  - path: HuggingFaceH4/llava-instruct-mix-vsft
    type: chat_template
    split: train[:1%]
```

| Key | Value | Why required |
|-----|-------|-------------|
| `processor_type` | `AutoProcessor` | Loads the model's multimodal processor (tokenizer + image/audio processor) |
| `skip_prepare_dataset` | `true` | Dataset preparation must be skipped; preprocessing happens per-batch |
| `remove_unused_columns` | `false` | Image and media columns must remain in the dataset during training |
| `sample_packing` | `false` | Not yet supported with multimodal models |

### Optional: LoRA on the language model only

When using LoRA with a VLM, it is recommended to freeze the vision encoder and adapter layers, training only the language model:

```yaml
adapter: lora
lora_target_modules: 'model.language_model.layers.[\d]+.(mlp|cross_attn|self_attn).(up|down|gate|q|k|v|o)_proj'
```

### Optional: image resizing

Resize images to a fixed size before passing them to the model processor:

```yaml
image_size: 512
image_resize_algorithm: bilinear
```

  <Accordion title="Gemma 4">
    All Gemma 4 variants (E2B, E4B, 26B-A4B, 31B) load as multimodal models — even for text-only training:

    ```yaml
    base_model: google/gemma-4-E2B-it  # or E4B-it, 26B-A4B, 31B

    chat_template: gemma4
    freeze_mm_modules: true  # freeze vision/audio encoders for text-only or vision LoRA

    lora_target_modules: 'model.language_model.layers.[\d]+.(_checkpoint_wrapped_module.)?(mlp|self_attn).(up|down|gate|q|k|v|o)_proj'
    ```

    For the 26B-A4B MoE variant, enable ScatterMoE and expert LoRA:

    ```yaml
    plugins:
      - axolotl.integrations.cut_cross_entropy.CutCrossEntropyPlugin
      - axolotl.integrations.kernels.KernelsPlugin
    use_kernels: true
    use_scattermoe: true
    experts_implementation: scattermoe

    lora_target_parameters:
      - experts.gate_up_proj
      - experts.down_proj
    ```

    <Warning>
    Gemma 4 VLM training starts with high loss (8–15). This is expected — see the training stability guide for details.
    </Warning>

    <Tip>
    For DDP training, axolotl auto-detects Gemma4 and sets `use_reentrant=False` and `ddp_find_unused_parameters=True`. When `activation_offloading: true`, use `freeze_mm_modules: true` instead. For FSDP2, use `fsdp_transformer_layer_cls_to_wrap: Gemma4TextDecoderLayer`.
    </Tip>
  </Accordion>

## Dataset format

Multimodal datasets use an extended `chat_template` format based on the OpenAI Messages API. Each message has a `role` and a `content` field, where `content` is a list of typed content blocks.

### Image inputs

For image loading, use `"type": "image"` within a content block, combined with one of these source keys:

```json
{"type": "image", "path": "/path/to/image.jpg"}
{"type": "image", "url": "https://example.com/image.jpg"}
{"type": "image", "base64": "..."}
{"type": "image", "image": "<PIL.Image>"}
```

<Note>
For backwards compatibility: if the dataset has an `images` or `image` column of type `list[Image]`, those images are automatically appended to the first content block. If `content` is a plain string, it is automatically converted to `[{"type": "text", "text": "..."}]`.
</Note>

### Audio inputs

```bash
pip3 install librosa==0.11.0
```

```json
{"type": "audio", "path": "/path/to/audio.mp3"}
{"type": "audio", "url": "https://example.com/audio.mp3"}
{"type": "audio", "audio": "<np.ndarray>"}
```

### Video inputs

<Warning>
Video support is not well tested at this time. Contributions are welcome.
</Warning>

```json
{"type": "video", "path": "/path/to/video.mp4"}
{"type": "video", "url": "https://example.com/video.mp4"}
{"type": "video", "video": "<np.ndarray | list[PIL.Image] | torch.Tensor>"}
```

### Full example dataset row

```json
[
  {
    "messages": [
      {
        "role": "system",
        "content": [
          {"type": "text", "text": "You are a helpful assistant."}
        ]
      },
      {
        "role": "user",
        "content": [
          {
            "type": "image",
            "url": "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/bee.jpg"
          },
          {"type": "text", "text": "Describe this image in detail."}
        ]
      },
      {
        "role": "assistant",
        "content": [
          {"type": "text", "text": "The image shows a bee on a flower."}
        ]
      }
    ]
  }
]
```

## Known limitations

- `sample_packing: false` is required — sample packing is not yet supported for multimodal models.
- Sequence length truncation and dropping is not applied, because each architecture processes non-text tokens differently.
- Some models require additional pip dependencies (see per-model notes above).

## Troubleshooting

**`PIL.UnidentifiedImageError: cannot identify image file`**

PIL could not retrieve the file at the specified `url`. Check for typos, or verify that the server is not blocking the request. Try using `path` or `base64` as an alternative.
