Axolotl makes it simple to go from a base model to a fine-tuned model in just a few commands. This guide walks through training a 1B Llama model with LoRA — small enough to run on most GPUs, but a complete end-to-end example of Axolotl's full workflow.

<Steps>
  <Step title="Install Axolotl">
    If you haven't installed Axolotl yet, see the [Installation guide](/installation). The quickest path:

    ```bash
    export UV_TORCH_BACKEND=cu128
    uv venv --python 3.12 && source .venv/bin/activate
    uv pip install --no-build-isolation axolotl[flash-attn,deepspeed]
    ```
  </Step>

  <Step title="Fetch example configs">
    Download a set of ready-to-use YAML configs for common models:

    ```bash
    axolotl fetch examples
    ```

    This creates an `examples/` directory with configs for Llama 3, Mistral, Qwen, and more.
  </Step>

  <Step title="Train with LoRA">
    Run training on the 1B Llama example — this uses LoRA so it fits on most GPUs:

    ```bash
    axolotl train examples/llama-3/lora-1b.yml
    ```

    Axolotl will download the base model, process the dataset, run the training loop, and save the LoRA adapter weights to `./outputs/lora-out`.
  </Step>

  <Step title="Run inference">
    Test your trained model interactively:

    ```bash
    axolotl inference examples/llama-3/lora-1b.yml --lora-model-dir="./outputs/lora-out"
    ```

    Or launch a Gradio web UI:

    ```bash
    axolotl inference examples/llama-3/lora-1b.yml --lora-model-dir="./outputs/lora-out" --gradio
    ```
  </Step>

  <Step title="Merge LoRA weights (optional)">
    Merge the LoRA adapter back into the base model for deployment:

    ```bash
    axolotl merge-lora examples/llama-3/lora-1b.yml --lora-model-dir="./outputs/lora-out"
    ```

    The merged model is saved to `./outputs/lora-out/merged`.
  </Step>
</Steps>

## Understanding the config file

Every Axolotl training run is controlled by a single YAML file. Here's the core of `examples/llama-3/lora-1b.yml`:

```yaml lora-1b.yml
base_model: NousResearch/Llama-3.2-1B

load_in_8bit: true
adapter: lora

datasets:
  - path: teknium/GPT4-LLM-Cleaned
    type: alpaca
dataset_prepared_path: last_run_prepared
val_set_size: 0.1
output_dir: ./outputs/lora-out
```

<Tip>
To switch training modes, change just two lines:
- **Full fine-tuning**: remove `load_in_8bit` and `adapter`
- **QLoRA**: replace with `load_in_4bit: true` and `adapter: qlora`
</Tip>

## Train on your own data

Create `my_training.yml` with your dataset:

```yaml my_training.yml
base_model: NousResearch/Llama-3.2-1B

load_in_8bit: true
adapter: lora

micro_batch_size: 2
num_epochs: 3
learning_rate: 0.0003

datasets:
  - path: my_data.jsonl
    type: alpaca
output_dir: ./outputs/my-model
```

Your `my_data.jsonl` should contain one JSON object per line in alpaca format:

```json
{"instruction": "Classify this text", "input": "I love this!", "output": "positive"}
{"instruction": "Classify this text", "input": "Not good at all", "output": "negative"}
```

Then train:

```bash
axolotl train my_training.yml
```

## Preprocess large datasets

For large datasets, preprocess once and reuse the tokenized cache to skip re-tokenization on every run:

```bash
axolotl preprocess my_training.yml
```

Set `dataset_prepared_path: last_run_prepared` in your config to save the cache.
