Axolotl can be installed in several ways depending on your environment and preferences. The recommended approach for most users is the **uv quick install**, which is fast, reliable, and handles CUDA version selection automatically. Docker is the easiest option if you want a fully pre-configured environment.

<Note>
  **Requirements**: NVIDIA GPU (Ampere architecture or newer for `bf16` and Flash Attention) or AMD GPU, Python ≥3.11, PyTorch ≥2.9.0.
</Note>

<Warning>
  For Blackwell GPUs, use PyTorch 2.9.1 and CUDA 12.8 (`cu128`).
</Warning>

## Installation methods

<Tabs>
  <Tab title="uv (recommended)">
    [uv](https://docs.astral.sh/uv/) is a fast Python package installer built in Rust. It is the primary recommended method for installing Axolotl.

    <Steps>
      <Step title="Install uv">
        If you don't already have uv installed, run:

        ```bash Install uv
        curl -LsSf https://astral.sh/uv/install.sh | sh
        source $HOME/.local/bin/env
        ```
      </Step>

      <Step title="Set your CUDA version">
        Choose the CUDA version that matches your system. Common options are `cu128` (CUDA 12.8) and `cu130` (CUDA 13.0):

        ```bash Set CUDA version
        export UV_TORCH_BACKEND=cu128  # or cu130
        ```
      </Step>

      <Step title="Create a virtual environment">
        ```bash Create venv
        uv venv --no-project --relocatable
        source .venv/bin/activate
        ```
      </Step>

      <Step title="Install Axolotl">
        Install Axolotl with Flash Attention and DeepSpeed extras:

        ```bash Install Axolotl
        uv pip install --no-build-isolation axolotl[flash-attn,deepspeed]
        ```

        <Tip>
          You can omit `flash-attn` or `deepspeed` if you don't need them. Flash Attention requires an Ampere or newer GPU. DeepSpeed is only needed for multi-GPU training with ZeRO.
        </Tip>
      </Step>

      <Step title="Fetch example configs">
        Download ready-to-use example YAML configs:

        ```bash Fetch examples
        axolotl fetch examples
        axolotl fetch deepspeed_configs  # optional
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Edge / dev build">
    To install from source and get the latest unreleased features:

    <Steps>
      <Step title="Clone the repository">
        ```bash Clone
        git clone https://github.com/axolotl-ai-cloud/axolotl.git
        cd axolotl
        ```
      </Step>

      <Step title="Set your CUDA version">
        ```bash Set CUDA version
        export UV_TORCH_BACKEND=cu128  # or cu130
        ```
      </Step>

      <Step title="Sync dependencies and install">
        `uv sync` creates a `.venv`, installs exact pinned versions from `uv.lock`, and sets up an editable install automatically:

        ```bash Sync
        uv sync --extra flash-attn --extra deepspeed
        source .venv/bin/activate
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Docker">
    Docker provides a fully pre-configured environment with all dependencies installed. This is the easiest way to get started without managing your own environment.

    <Steps>
      <Step title="Run the latest image">
        ```bash Docker run
        docker run --gpus '"all"' --rm -it --ipc=host axolotlai/axolotl-uv:main-latest
        ```
      </Step>

      <Step title="Optional: Advanced configuration">
        For persistent storage and HuggingFace cache mounting:

        ```bash Docker advanced
        docker run --privileged --gpus '"all"' --shm-size 10g --rm -it \
          --name axolotl --ipc=host \
          --ulimit memlock=-1 --ulimit stack=67108864 \
          --mount type=bind,src="${PWD}",target=/workspace/axolotl \
          -v ${HOME}/.cache/huggingface:/root/.cache/huggingface \
          axolotlai/axolotl-uv:main-latest
        ```
      </Step>
    </Steps>

    <Warning>
      For Blackwell GPUs, use `axolotlai/axolotl-uv:main-py3.11-cu128-2.9.1` or the cloud variant `axolotlai/axolotl-cloud-uv:main-py3.11-cu128-2.9.1`.
    </Warning>

    </Tip>

  <Tab title="pip (alternative)">
    If you cannot use uv, you can install with pip directly. Make sure PyTorch is installed first.

    <Steps>
      <Step title="Install PyTorch">
        Follow the instructions at [pytorch.org/get-started/locally](https://pytorch.org/get-started/locally/) to install the correct version for your CUDA toolkit.
      </Step>

      <Step title="Install Axolotl">
        ```bash Install with pip
        pip3 install -U packaging setuptools wheel ninja
        pip3 install --no-build-isolation axolotl[flash-attn,deepspeed]
        ```
      </Step>

      <Step title="For editable installs (development)">
        ```bash Editable install
        pip3 install -U packaging setuptools wheel ninja
        pip3 install --no-build-isolation -e '.[flash-attn,deepspeed]'
        ```
      </Step>
    </Steps>

## Platform-specific notes

### AMD GPU / ROCm

AMD GPUs are supported. Use a ROCm-compatible PyTorch build and ensure ROCm drivers are installed. Refer to the [PyTorch ROCm installation guide](https://pytorch.org/get-started/locally/) for the correct install command for your ROCm version.

### macOS

```bash macOS install
uv pip install --no-build-isolation -e '.'
```

Flash Attention and DeepSpeed are not supported on macOS. Training on Apple Silicon uses MPS, which has some limitations compared to CUDA.

### Windows

<Warning>
  Native Windows installation is not officially supported. Use WSL2 (Windows Subsystem for Linux) or Docker for a supported environment.
</Warning>

## Migrating from pip to uv

If you have an existing pip-based Axolotl installation:

```bash Migrate to uv
# Install uv
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env

# Set CUDA version and create a fresh venv
export UV_TORCH_BACKEND=cu128  # or cu130
uv venv --no-project --relocatable
source .venv/bin/activate

# Reinstall axolotl
uv pip install --no-build-isolation axolotl[flash-attn,deepspeed]
```
