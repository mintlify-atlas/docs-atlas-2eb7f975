Axolotl is a free and open-source tool designed to streamline post-training and fine-tuning for large language models (LLMs). It unifies the entire pipeline — dataset preprocessing, training, evaluation, and inference — into a single YAML configuration file, so you spend less time on boilerplate and more time on results.

<Note>
  Axolotl requires an NVIDIA GPU (Ampere or newer for `bf16` and Flash Attention) or an AMD GPU, Python ≥3.11 (3.12 recommended), and PyTorch ≥2.9.1.
</Note>

## Key features

### Multiple model support

Train models like Llama, Mistral, Mixtral, Qwen, Pythia, and hundreds more from the Hugging Face Hub. Recent additions include Mistral Small 4, Qwen3.5, GLM-4.7-Flash, and Granite 4.

### Training methods

Axolotl supports the full spectrum of fine-tuning approaches:

| Method | Config key | Best for |
|--------|-----------|----------|
| Supervised Fine-Tuning (SFT) | *(default, no `rl:` key)* | Instruction tuning, task adaptation |
| DPO / IPO | `rl: dpo` / `rl: ipo` | Paired preference alignment |
| KTO | `rl: kto` | Unpaired binary preference labels |
| ORPO | `rl: orpo` | Single-stage alignment without a ref model |
| GRPO | `rl: grpo` | RL with verifiable reward functions |
| Reward Modelling | `rl: reward_trainer` | Training a reward signal for RL |

### Multimodal training

Fine-tune vision-language models (VLMs) including Llama 4, Qwen2-VL, Qwen3-VL, Gemma 3n, InternVL 3.5, and audio models like Voxtral — with image, video, and audio support.

### Performance optimizations

- **Attention**: Flash Attention 2/3/4, Xformers, Flex Attention, SageAttention
- **Kernels**: Liger Kernel, Cut Cross Entropy, ScatterMoE for MoE models
- **Packing**: Multipacking for efficient sequence batching
- **Parallelism**: FSDP1, FSDP2, DeepSpeed ZeRO, Sequence Parallelism, Tensor Parallelism
- **Adapters**: LoRA optimizations for reduced VRAM and faster training

### Flexible dataset handling

Load datasets from local files, the Hugging Face Hub, or cloud storage (S3, Azure Blob, GCS, OCI). Supports alpaca, chat template, input/output, completion, and preference pair formats.

### Cloud-ready deployment

Axolotl ships Docker images and PyPI packages for use on cloud platforms. One-click templates are available on RunPod, Vast.ai, PRIME Intellect, Modal, Novita, JarvisLabs, and Latitude.sh.

## How it works

Every Axolotl training run is defined by a single YAML configuration file. The same file drives dataset preprocessing, training, evaluation, and inference — no code changes needed to switch between methods or adapters.

```yaml Example config
base_model: NousResearch/Llama-3.2-1B
adapter: lora

datasets:
  - path: teknium/GPT4-LLM-Cleaned
    type: alpaca

output_dir: ./outputs/lora-out
```

Run training with:

```bash Train
axolotl train config.yml
```

And inference with the same file:

```bash Inference
axolotl inference config.yml --lora-model-dir="./outputs/lora-out"
```

## AI agent support

Axolotl ships with built-in documentation optimized for AI coding agents (Claude Code, Cursor, Copilot, and others). These docs are bundled with the pip package — no repository clone needed.

```bash Agent docs
# Show overview and available training methods
axolotl agent-docs

# Topic-specific references
axolotl agent-docs sft
axolotl agent-docs grpo
axolotl agent-docs preference_tuning
axolotl agent-docs --list
```
